## What was built

The MCP (Model Context Protocol) server exposes Sovereign CRM data and actions as tools
that any MCP-compatible AI agent (Claude, Cursor, Copilot, etc.) can call at runtime.

### Architecture

```
POST /api/mcp  (authenticated, JSON-RPC 2.0)

  tools/list   →  returns all tool definitions + JSON Schemas
  tools/call   →  executes a tool with arguments, returns JSON result
  resources/list → returns resources (stub, ready for future use)
```

### Key design decisions

1. **Embedded in main API** — Reuses existing auth middleware, pgx pool, and chi router.
   No separate binary or port needed. Accessible at `/api/mcp` alongside all other endpoints.

2. **JSON-RPC 2.0 over HTTP POST** — Standard MCP protocol. Request:
   ```json
   {"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"get_pipeline_summary","arguments":{}}}
   ```
   Response wraps in MCP content format:
   ```json
   {"jsonrpc":"2.0","id":1,"result":{"content":[{"type":"json","json":{...}}]}}
   ```

3. **Row-level security** — All queries filter by `tenant_id` from auth context.
   No bypass possible; shares the same middleware chain.

4. **pgx native uuid.UUID** — User/tenant IDs passed as `uuid.UUID` (not strings)
   and used directly in parameterized queries.

5. **No new tables** — Zero migrations. Reads existing contacts, deals, leads,
   pipeline_stages, activities, organizations tables.

### Tool schema format

Each tool returns a JSON Schema input schema so AI clients can generate arguments:
```json
{
  "name": "create_contact",
  "description": "Create a new CRM contact",
  "inputSchema": {
    "type": "object",
    "properties": {
      "first_name": {"type": "string", "description": "First name (required)"},
      "last_name": {"type": "string", "description": "Last name (required)"},
      ...
    },
    "required": ["first_name", "last_name"]
  }
}
```

## How to test

```bash
# List all tools
curl -X POST http://localhost:8080/api/mcp \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'

# Call a tool
curl -X POST http://localhost:8080/api/mcp \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{"name":"get_pipeline_summary","arguments":{}}}'
```
