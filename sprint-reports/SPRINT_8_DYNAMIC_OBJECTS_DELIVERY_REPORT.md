## What was built

Sprint 8 polished the existing Dynamic Object Builder (from Sprint 5) and closed the remaining UX and integration gaps.

### Gap 1: Record List showed raw JSON (FIXED)

**`web/app/objects/[objectId]/page.tsx`** — Complete rewrite.

Before: Showed truncated JSON string (`{"name": "Acme", "indus...`) in a single column. Unusable for real data.

After: Renders dynamic columns based on the object type's field definitions. Each field type has its own renderer:
- `text`, `number`, `currency` — formatted values
- `boolean` — check/X icons
- `date` — localized date format
- `select`/`multi_select` — comma-joined chip display
- `url` — clickable link
- `percent` — formatted with `%` suffix

Also added: inline search bar, row click navigation to detail page.

### Gap 2: No record detail/edit page (CREATED)

**`web/app/objects/[objectId]/[recordId]/page.tsx`** — New page.

Displays a full dynamic form pre-populated with existing data (reuses the `renderField` component from the new-record page). Supports:
- Edit all field values with proper input types per field schema
- Save changes via `PUT /dynamic-objects/{id}/records/{recordId}`
- Delete record with confirmation
- Back navigation to record list
- Disable save button when no changes detected

### Gap 3: MCP tools missing for Dynamic Objects (ADDED)

**`api/internal/mcp/tools_dynamic_objects.go`** — 6 new MCP tools:

| Tool | Description |
|------|-------------|
| `list_dynamic_object_types` | List all custom object definitions |
| `get_dynamic_object_type` | Get definition + field schema |
| `list_dynamic_records` | List records of a type |
| `get_dynamic_record` | Get single record with data |
| `create_dynamic_record` | Create record with field values |
| `search_dynamic_records` | Search across all types |

Wired into `api/internal/mcp/server.go` via `registerDynamicObjectTools()` call in `NewServer`.
