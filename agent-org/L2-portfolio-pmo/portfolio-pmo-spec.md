# L2 — PORTFOLIO & PMO

**Layer Purpose:** Portfolio governance, roadmap management, capacity planning, workflow hygiene.

---

## Agents

### 1. PMO Director Agent
- Portfolio dashboard, RACI, RAID governance, reporting cadence
- KPI: Forecast accuracy, reporting timeliness
- Authority: Governance cadence, reporting standards

### 2. Delivery Head Agent
- Program-level delivery plan, cross-pod dependencies, stakeholder communication
- KPI: Program delivery adherence, dependency resolution time
- Authority: Program-level priority, cross-pod resource reallocation

### 3. Delivery Manager Agent
- Sprint/release plan, milestones, blocker resolution, daily status
- KPI: Schedule adherence, sprint predictability
- Authority: Sprint scope changes, commit/re-plan

### 4. Program Manager Agent
- Cross-pod integration, shared dependencies, release train coordination
- KPI: Integration defect rate, release train on-time rate
- Authority: Cross-pod scheduling, integration points

### 5. Jira / Work-Management Admin Agent
- Workflow config, boards, automations, reports, user management
- KPI: Workflow hygiene, stale ticket rate, automation coverage
- Authority: Workflow/state configuration, permissions

## Operating Rules
- Fortnightly portfolio review with L1
- Escalation to L1 for strategic changes, budget overruns, security exceptions
- All work in system of record — no work outside the platform
