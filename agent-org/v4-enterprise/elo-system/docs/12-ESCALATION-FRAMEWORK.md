# ELO DOCUMENT 12: ESCALATION FRAMEWORK
# Enterprise Learning Operations — When Things Go Wrong

---

## Escalation Levels

### Level 1: Operational (Tier 3 → Tier 2)
**Trigger:** Individual agent issues
**Response Time:** Same day
**Examples:**
- Agent missed a learning cycle
- Agent submitted low-quality self-report
- Learning pack delivered with low relevance score
- Source quality degradation detected

**Actions:**
1. Tier 3 feeder notifies Tier 2 lead
2. Tier 2 lead investigates root cause
3. Tier 2 lead takes corrective action
4. Tier 2 lead documents in domain summary

### Level 2: Domain (Tier 2 → Tier 1)
**Trigger:** Domain-wide patterns
**Response Time:** Within 48 hours
**Examples:**
- 3+ agents in domain stagnating
- Domain learning score declining for 5+ days
- Certification progress frozen for 7+ days
- Cross-domain learning transfer failing
- Tier 3 feeders delivering stale content

**Actions:**
1. Tier 2 lead escalates to Tier 1 director
2. Tier 1 director conducts domain health check
3. Tier 1 director approves intervention plan
4. Tier 2 lead executes intervention
5. Tier 1 director monitors for 7 days

### Level 3: Enterprise (Tier 1 → Founder)
**Trigger:** Enterprise-wide systemic issues
**Response Time:** Within 72 hours
**Examples:**
- 3+ domains showing simultaneous decline
- L&D system architecture failure
- Enterprise learning score dropping below 50
- L&D system disconnected from 10%+ of agents
- Source quality enterprise-wide degradation
- Certification ROI turning negative

**Actions:**
1. Tier 1 director escalates to Founder
2. Founder conducts enterprise L&D review
3. Tier 1 council develops remediation plan
4. Founder approves resources for remediation
5. Tier 2-25 (L&D Evolution) implements fixes
6. Tier 1 council monitors for 30 days

## Stagnation Detection

### Signals
- Same challenge reported 3+ days without change
- Quality delta negative for 3+ consecutive days
- Learning score flat (no change) for 7+ days
- Certification readiness score declining
- Self-report quality declining (less detail over time)

### Response Protocol
1. **Day 1:** Tier 2 lead notes stagnation signal
2. **Day 2:** Tier 2 lead investigates root cause
3. **Day 3:** Tier 2 lead implements intervention
4. **Day 5:** Tier 2 lead checks if intervention working
5. **Day 7:** If still stagnant, escalate to Tier 1
6. **Day 14:** If still stagnant, escalate to Founder

---

## Disconnection Detection

An agent is considered "disconnected" from ELO if:
- No self-report submitted for 48+ hours
- No learning pack consumed for 72+ hours
- No learning applied for 5+ days
- Agent status set to "inactive" by Tier 2

### Reconnection Protocol
1. Tier 2 lead attempts to contact agent
2. Tier 2 lead verifies agent is operational
3. If agent is operational but not learning: investigate and intervene
4. If agent has been restructured: re-map to new Tier 2 lead
5. If agent has been deactivated: remove from ELO mapping
6. Document reconnection in agent profile
