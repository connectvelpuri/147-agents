## Scaling Triggers

### Trigger 1: New Product Line Added
**Example:** Adding a Supply Chain product

**ELO Response:**
1. Create new Tier 2 Domain Lead for Supply Chain
2. Create 2-3 Tier 3 feeders for Supply Chain domain
3. Map all new Supply Chain agents to new Tier 2 lead
4. Add Supply Chain to appropriate cluster (Cluster 4: Business Products)
5. Connect to cross-domain learning flows
6. Notify Tier 2-24 (Emerging) for technology radar update
7. Onboard within 24 hours

**Capacity Impact:** +1 Tier 2 lead, +2-3 Tier 3 feeders, +N operational agents

### Trigger 2: New Domain Emerges (e.g., Quantum Computing)
**ELO Response:**
1. Tier 2-24 (Emerging) identifies new domain
2. Tier 1-05 (Innovation) approves domain addition
3. Create new Tier 2 Domain Lead
4. Create 2-3 Tier 3 feeders
5. Identify relevant existing agents to connect
6. Establish domain-specific knowledge sources
7. Create domain learning curriculum

**Capacity Impact:** +1 Tier 2 lead, +2-3 Tier 3 feeders, existing agents re-mapped

### Trigger 3: Agent Count Grows by 100+
**ELO Response:**
1. Tier 2-25 (L&D Evolution) detects scaling pressure
2. Assess if existing Tier 2 leads need help
3. If a Tier 2 lead has >30 agents: consider splitting domain
4. If Tier 3 feeders overwhelmed: add more feeders
5. Monitor learning pack quality under load
6. Adjust scan frequency if needed

**Capacity Impact:** May require additional Tier 3 feeders per domain

### Trigger 4: Enterprise Grows to 1000+ Agents
**ELO Response:**
1. Tier 1 Council reviews ELO architecture
2. Consider adding sub-Tier 2 positions (Tier 2a: regional leads)
3. Consider adding more Tier 3 feeders per domain
4. Consider adding domain-specific Tier 1 directors
5. Increase automation and reduce manual curation
6. Enhance source diversity and quality

**Capacity Impact:** May require Tier 2a layer or expanded Tier 1

## Scaling Principles

1. **Scale the bottom first** — add Tier 3 feeders before Tier 2 leads
2. **Split domains when overloaded** — >30 agents per Tier 2
3. **Never compromise the 3 cycles** — frequency stays constant
4. **Never reduce source quality** — quality stays constant
5. **Always maintain cross-domain connections** — clusters grow with domains
6. **Always onboard within 24 hours** — speed stays constant
7. **Always keep founder visible** — daily pulse never stops
