# MANDATORY CADENCES & ARTIFACTS

---

## Cadences

### Daily
- **Pod Sync** (15 min) — What I did, what I'll do, what's blocking me
  - Attendees: All pod members
  - Artifact: Updated ticket statuses

### Twice Weekly
- **Dependency Review** (30 min, Tue+Thu)
  - Attendees: All Pod Leads + Program Manager
  - Artifact: Updated dependency map in RAID log

### Weekly
- **Architecture Review Board** (60 min, Wed)
  - Attendees: EA, SA, CTO, SecEng, DevOps Lead
  - Artifact: ADRs updated, exceptions documented

- **Quality Review** (45 min, Thu)
  - Attendees: QA Lead, SQA, EM, SecEng
  - Artifact: Quality dashboard, defect log

### Fortnightly
- **Portfolio Review** (90 min, alternate Mon)
  - Attendees: Executive Council + L2 agents
  - Artifact: Portfolio dashboard, RAID log, capacity plan

### Monthly
- **Operating Review** (120 min, first Mon)
  - Attendees: Full Executive Council + L2 + CoE Leads
  - Artifact: Monthly operating report, OKR tracker

### Quarterly
- **Sprint Retrospective / Improvement Review** (120 min)
  - Attendees: All CoE Leads + Continuous Improvement
  - Artifact: Improvement backlog, process change log

---

## Mandatory Artifacts

### Per-Ticket
- Business context (PM) — PRD reference + user story
- Technical design (SA) — HLD/LLD before coding
- Test evidence (QA Lead) — test report + coverage
- Release evidence (RM) — checklist + approval
- Operational owner (Pod Lead) — named agent

### Per-Sprint
- Sprint plan, sprint board, burndown, review, retro

### Per-Feature
- PRD, architecture design, ADR, test plan, API spec, design specs, runbook, user docs

### Per-Release
- Release plan, release notes, monitoring plan, rollback plan, post-release review

---

## Quality Standards
1. Complete — no placeholders
2. Current — updated within 24h
3. Accessible — in knowledge base, searchable
4. Reviewable — peer reviewed, version controlled
5. Actionable — contains decisions and next steps
