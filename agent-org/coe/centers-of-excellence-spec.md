# CENTERS OF EXCELLENCE (CoEs)

**Version:** 1.0

---

## Dual-Reporting Model
- Functional reporting → CoE Lead (standards, quality, capability)
- Delivery reporting → Pod Lead (execution, delivery, sprint)
- Conflict resolution: Pod Lead decides priority; CoE Lead decides quality bar

---

## The 9 CoEs

### 1. Product CoE
**Lead:** Product Director (L3)
**Members:** Product Manager, Business Analyst, UX Research
**Scope:** Product strategy, value framework, roadmap governance, discovery methodology
**Standards:** PRD template, epic/story format, acceptance criteria, prioritization framework
**Authority:** All product work follows Product CoE standards

### 2. Design CoE
**Lead:** UX Design Lead (L3)
**Members:** UI/UX Designer
**Scope:** UX standards, design system, accessibility (WCAG 2.1 AA), research quality
**Standards:** Design system docs, component specs, wireframe standards, accessibility checklist
**Authority:** All UI uses design system; UX Lead vetoes non-accessible designs

### 3. Architecture CoE
**Lead:** Enterprise Architect (L4)
**Members:** Solution Architect, Platform Architect
**Scope:** Reference architecture, standards, reusable patterns, technology radar, ARB
**Standards:** ADR template, C4 model, integration patterns, API/data/security architecture
**Authority:** All architectural decisions documented as ADRs; Enterprise Architect chairs ARB

### 4. Engineering CoE
**Lead:** Engineering Manager (L4)
**Members:** Senior Software Engineer, Senior Frontend, Senior Backend
**Scope:** Coding standards, code review, frameworks, developer productivity, shared libs
**Standards:** Code style guide, review checklist, PR template, testing standards, git strategy
**Authority:** All code meets standards; code review mandatory; tech debt tracked

### 5. Data & AI CoE
**Lead:** Data Scientist (L4) + AI Engineer (L4)
**Members:** Data Engineer, Applied Scientist
**Scope:** Data pipelines, ML/AI development, experimentation, data governance, LLM evaluation
**Standards:** Pipeline design, ML evaluation criteria, experiment methodology, AI safety
**Authority:** All AI features pass evaluation before production; guardrails mandatory

### 6. Quality CoE
**Lead:** QA Lead (L5)
**Members:** Senior QA Engineer
**Scope:** STLC management, test automation, quality gates, defect prevention
**Standards:** Test strategy, test case format, automation framework, defect classification
**Authority:** QA Lead vetoes release readiness; test plans before coding; quality metrics weekly

### 7. Platform & SRE CoE
**Lead:** DevOps Lead (L5) + SRE Lead (L5)
**Members:** DevOps Engineer, Junior DevOps
**Scope:** CI/CD, IaC, observability, incident response, capacity planning
**Standards:** CI/CD template, IaC modules, monitoring standards, deployment strategies
**Authority:** All deployments through CI/CD; DevOps Lead approves infra changes; SRE defines SLOs

### 8. Security & Compliance CoE
**Lead:** CISO/Compliance Head (L1)
**Members:** Security Engineer (L5)
**Scope:** IAM, security policy, vulnerability management, audit, compliance (SOC2/GDPR/HIPAA)
**Standards:** Security policy, OWASP checklist, vulnerability process, data classification
**Authority:** Security Engineer vetoes releases with critical vulns; CISO approves exceptions

### 9. Delivery Excellence CoE
**Lead:** Delivery Head (L2)
**Members:** PMO Director, Delivery Manager, Program Manager
**Scope:** Delivery governance, RAID management, metrics, escalation, sprint best practices
**Standards:** Sprint template, standup format, review format, retro format, RAID format
**Authority:** All delivery follows Delivery Excellence standards; cadence discipline enforced
