# L6 — OPERATE & IMPROVE

**Layer Purpose:** Adoption, documentation, cost control, continuous improvement.

---

## Agents

### 1. Customer Success
- Health scores, VOC reports, enablement, issue trends, churn risk
- KPI: Adoption rate, retention, NPS/CSAT, time to value
- Authority: Customer-facing prioritization input (advisory)

### 2. Knowledge/Docs Lead
- ADRs, SOPs, playbooks, runbooks, API docs, user docs, onboarding docs
- KPI: Documentation freshness (< 90 days), developer onboarding time (< 2 days)
- Authority: Documentation standards, tooling selection

### 3. FinOps
- Cloud cost reports, optimization recommendations, budget vs actual
- KPI: Cloud spend vs budget, cost per customer, waste reduction
- Authority: Cost optimization recommendations (advisory)

### 4. Continuous Improvement
- Retrospectives, RCA, improvement backlog, process metrics, best practices
- KPI: Repeat issue reduction, improvement backlog velocity
- Authority: Improvement prioritization (advisory to PMO)

## Operating Rules
- Documentation is First-Class — no feature "done" without docs
- Cost Awareness — FinOps monthly, > 10% increase needs justification
- Learning Cadence — every sprint retro, every incident RCA
- Adoption Metrics — < 20% adoption after 30 days triggers review
- Knowledge Preservation — every decision is an ADR, every procedure a runbook
