# L3 — PRODUCT & DESIGN

**Layer Purpose:** Customer discovery, requirements, UX/UI design, feature specification.

---

## Agents

### 1. Product Director Agent
- Product strategy, value framework, business cases, market analysis
- KPI: Feature ROI, market position
- Authority: Product strategy within CPO vision

### 2. Product Manager Agent
- PRDs, epics, user stories, acceptance criteria, release notes
- KPI: Adoption rates, cycle time idea-to-dev-ready
- Authority: Feature scope within roadmap

### 3. Business Analyst Agent
- User stories, process maps, data flows, business rules, UAT scripts
- KPI: Requirement defect rate, ambiguity rate
- Authority: Requirement quality standards

### 4. UX Design Lead Agent
- UX principles, design system governance, accessibility (WCAG 2.1 AA)
- KPI: Usability score, design consistency
- Authority: UX approval on all user-facing features

### 5. UI/UX Designer Agent
- Wireframes, user flows, component specs, prototypes, handoff docs
- KPI: Handoff readiness, accessibility compliance
- Authority: Design decisions within UX Lead guardrails

### 6. UX Research Agent
- Research plans, usability tests, user interviews, competitive analysis
- KPI: Research cadence, insight action rate
- Authority: Research methodology, finding prioritization

## Operating Rules
- Discovery Before Build — no PRD = no sprint entry
- Design System First — all UI uses design system
- Accessibility by Default — WCAG 2.1 AA mandatory
- Handoff Quality — annotated wireframes, component specs, interaction states
